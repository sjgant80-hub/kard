peer-to-peer duel · no server

Duel anyone, anywhere.

WebRTC DataChannel · STUN-only · zero infrastructure between you and your opponent. Manually exchange offer + answer codes (paste them anywhere · DM, Discord, LinkedIn comment, carrier pigeon). Once connected, moves sync directly browser-to-browser. Both peers maintain an audit chain so every match is mutually verifiable.

1 · Host a duel

Create an offer code. Send to friend. Wait for their answer. Paste it. Game begins.

2 · Join a duel

3 · Start the duel

Both peers connected. Host starts the match. Each move syncs in real time, every move is hashed by both sides independently — the chains must match.

Caveats · honesty

Some NATs block direct WebRTC without a TURN relay · in those cases the connection won't open without a relay server we'd have to host. STUN-only works for ~80% of consumer networks · ~20% will need a v2.1 with optional TURN.

The audit chain is computed independently on both sides · if they diverge, someone is cheating. The chain check is part of the trophy verification.
